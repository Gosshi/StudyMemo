# REST

## REST とは

REST とはクライアント・サーバから発生したアーキテクチャである  
厳密にはクライアント・サーバアーキテクチャにいくつかの制約を加えたもの  
REST は Web 全体のアーキテクチャスタイルであり、守ることが重要である

## リソース

REST における重要な概念の一つである

Web におけるリソース: Web 上に存在する名前を持ったありとあらゆる情報  
→ つまり、URI を指す

まとめると、

- リソースとは Web 上の情報
- 世界中のリソースはそれぞれ URI で一意の名前を持つ
- URI を用いることで、プログラムはリソースが表現する情報にアクセスできる
  - URI でリソースを簡単に指し示せる性質のことを「アドレス可能性」と呼ぶ

## ステートレスサーバ

ステートレス：クライアントのアプリケーション状態をサーバで管理しないこと -

- 利点
  - サーバの実装が楽
  - クライアントからのリクエストに答えたあとすぐに計算リソースを開放できる

ただし、実際はステートレスでないサービスや API が多い

- 代表的な例としては Cookie を用いたセッション管理
- REST 的な視点から見ると、間違った HTTP の拡張
  - Cookie を使ったフォーム認証とか

## キャッシュ

キャッシュとはリソースの鮮度に基づいて、一度取得したリソースをクライアント側で使い回す方式  
通信回数と量が減る

## その他

- 統一インターフェース：インターフェースを固定
- 階層化システム：システムを階層に分離
- コードオンデマンド：プログラムをクライアントにダウンロードして実行(JavaScript)
